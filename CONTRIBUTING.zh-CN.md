# 参与贡献 AIR

[English](CONTRIBUTING.md)

感谢你关注 AIR。

AIR 仍处于早期阶段，架构、实现、文档、测试和社区建设方面的贡献都非常有价值。

## 贡献方式

- 提交 issue，反馈 bug、缺口或设计问题
- 提出架构改进建议
- 实现 `session`、`vm`、`store`、`guest agent` 等模块
- 改进文档、示例和 onboarding 体验
- 参与公开 roadmap 和设计讨论

## 开始前建议

- 阅读 [README.md](README.md)
- 阅读 [ROADMAP.md](ROADMAP.md)
- 阅读 [`docs/`](docs) 下的产品和架构文档

## 贡献规则

- 保持改动聚焦且易审查
- 小步提交，避免无关改动打包在一起
- 遵循 [AGENTS.md](AGENTS.md) 中的仓库结构和命名规则
- 面向用户的文档必须同时提供中文和英文版本
- 修改任意一份对照文档时，要在同一提交中同步更新另一份
- 架构或公开行为变化时同步更新文档
- 可执行路径变化时补测试

## 推荐流程

1. 对非小改动先开 issue 或讨论
2. 从 `main` 派生分支
3. 做聚焦改动
4. 更新相关文档
5. 提交带上下文的 PR

## PR 期望

建议包含：

- 改了什么
- 为什么改
- 范围和权衡
- 若涉及代码，附测试证据
- 若是部分交付，说明后续工作

## 适合入门的方向

- CLI 骨架
- session 状态存储
- VM 生命周期封装
- guest agent 协议草案
- 文档整理和示例补充
- roadmap issue 拆分

## 社区方向

AIR 希望成为围绕以下主题开放演进的技术项目：

- AI 沙箱
- VM 级隔离
- agent runtime 基础设施
- 安全代码执行
