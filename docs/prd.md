# AIR 产品说明书

## 1. 产品简介

AIR（Agent Isolation Runtime）是一个为 AI Agent 提供安全执行环境的基础产品。它通过轻量虚拟机而不是共享内核容器来运行不可信代码，降低逃逸风险，并提供默认隔离、临时销毁和状态可控的执行能力。

## 2. 产品愿景

为 AI Agent 提供一个“默认安全的执行环境”，让代码执行从能力补丁变成标准基础设施。

## 3. 目标用户

- AI Coding Agent 平台
- 代码执行沙箱服务提供方
- 安全敏感的自动化研发系统
- 需要长会话执行环境的开发工具

## 4. 核心问题

当前 AI 生成代码通常运行在容器或宿主环境中，但这些环境默认是“运行可信代码”的设计。AIR 解决的问题是：当执行代码默认不可信时，如何提供更强的资源隔离和更低的宿主机风险。

## 5. 典型使用场景

### 5.1 一次性执行

用户提交脚本后直接运行，执行完成后销毁环境。

示例：

```bash
air run hello.py
```

### 5.2 有状态会话执行

用户创建 session，在同一 VM 内连续执行多条命令，保留文件系统状态。

示例：

```bash
air session create
air session exec <id> "echo hello > a.txt"
air session exec <id> "cat a.txt"
air session delete <id>
```

## 6. 功能需求

### 6.1 Run

- 创建一次性 VM
- 注入任务并执行
- 返回 stdout/stderr/exit_code
- 执行完成自动销毁

### 6.2 Session

- 创建 session
- 在指定 session 中执行命令
- 保持文件状态
- 删除 session 并回收资源

### 6.3 隔离控制

- 默认无网络
- 支持 CPU、内存、执行超时限制
- 后续支持白名单网络访问

### 6.4 环境管理

- 基础镜像只读
- 每个 session 独立 overlay
- 后续支持 snapshot/restore

### 6.5 生命周期管理

- 状态流转：创建、运行、空闲、停止、删除
- 闲置自动回收
- 异常 session 自动清理

## 7. 非功能需求

- 安全性：不共享业务环境，默认最小权限
- 可复现：同一基础镜像下执行环境一致
- 稳定性：通信失败、超时、Guest 崩溃可感知
- 可观测：记录命令、输出、状态与耗时

## 8. MVP 边界

MVP 只做单机、CLI、本地状态、文件通信、默认无网络，不做多租户、分布式调度、完整快照、Web UI。

## 9. 成功指标

- 成功执行率
- session 创建时延
- exec 平均时延
- 资源回收成功率
- 安全隔离缺陷数
